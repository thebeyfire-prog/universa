Authentication

Account KYC approval is the account approval for API access. After it is active, sign every server request with HMAC headers. Mutating requests should include an idempotency-key. The canonical signature payload is timestamp, nonce, method, path, and SHA-256 body hash. 

Headers:
content-type: application/json
idempotency-key: unique-operation-key
x-universa-api-key: $UNIVERSA_API_KEY
x-universa-timestamp: $TIMESTAMP_MS
x-universa-nonce: $NONCE
x-universa-signature: $SIGNATURE

Canonical string:
$TIMESTAMP_MS + "\n" +
$NONCE + "\n" +
$METHOD + "\n" +
$PATH_WITH_QUERY + "\n" +
sha256($RAW_BODY)

Signature:
hex(hmac_sha256($UNIVERSA_API_SECRET, canonical_string))

                    WEBHOOK
                    
Outbound events

                  

                  
                

                
Configure endpoints in the Webhooks panel. Universa sends signed JSON events for customer, KYC, virtual account, quote, transfer, and test events.

                

                  
Headers
x-universa-event-id, x-universa-event-type, x-universa-timestamp, x-universa-signature

                  
Signature
v1 = HMAC-SHA256(secret, timestamp + "." + raw_body)

                  
Retries
2xx marks delivered; failures retry with backoff and then dead-letter

                

                
const crypto = require("crypto")

function verifyUniversaWebhook(rawBody, headers, secret) {
  const timestamp = headers["x-universa-timestamp"]
  const signature = String(headers["x-universa-signature"] || "").replace(/^v1=/, "")
  const expected = crypto
    .createHmac("sha256", secret)
    .update(`${timestamp}.${rawBody}`)
    .digest("hex")

  return crypto.timingSafeEqual(
    Buffer.from(signature, "hex"),
    Buffer.from(expected, "hex")
  )
}
