Authentication

Account KYC approval is the account approval for API access. After it is active, sign every server request with HMAC headers. Mutating requests should include an idempotency-key. The canonical signature payload is timestamp, nonce, method, path, and SHA-256 body hash. 

Headers:
content-type: application/json
idempotency-key: unique-operation-key
x-universa-api-key: $UNIVERSA_API_KEY
x-universa-timestamp: $TIMESTAMP_MS
x-universa-nonce: $NONCE
x-universa-signature: $SIGNATURE

Canonical string:
$TIMESTAMP_MS + "\n" +
$NONCE + "\n" +
$METHOD + "\n" +
$PATH_WITH_QUERY + "\n" +
sha256($RAW_BODY)

Signature:
hex(hmac_sha256($UNIVERSA_API_SECRET, canonical_string))
POST

/v1/customers

Create an individual or business customer record. Money movement stays locked until that customer completes hosted KYC.

Scope
customers:write
Body
external_id, type, full_name, email, country_code, metadata
Returns
customer.id and status
curl -X POST "$UNIVERSA_API_URL/v1/customers" \
  -H "content-type: application/json" \
  -H "idempotency-key: customer-acme-001" \
  -H "x-universa-api-key: $UNIVERSA_API_KEY" \
  -H "x-universa-timestamp: $TIMESTAMP_MS" \
  -H "x-universa-nonce: $NONCE" \
  -H "x-universa-signature: $SIGNATURE" \
  -d '{
    "external_id": "user_123",
    "type": "individual",
    "full_name": "Jane Customer",
    "email": "jane@example.com",
    "country_code": "US",
    "metadata": { "plan": "sandbox" }
  }'
